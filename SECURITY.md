# Security Policy

## Supported Scope

Signatury is intended for on-prem Windows and Classic Outlook environments. Security reports are especially helpful for:

- code execution risks
- unsafe registry handling
- insecure file handling
- unintended disclosure of user or directory data
- template processing issues that could affect integrity or confidentiality

## Reporting a Vulnerability

Please do not open a public issue for sensitive vulnerabilities.

Until a dedicated security contact is configured, report issues privately to the maintainer through the repository hosting platform's private reporting feature if available.

When reporting, include:

- affected file or function
- impact
- reproduction steps
- suggested mitigation if known
